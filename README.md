# RA Record

RA（Research Assistant）工作记录仓库，用于记录每日工作内容、任务进展、遇到的困难与次日计划。

## 目录结构

```
RA_Record/
├── README.md
└── <月份>/           # 按月份分文件夹，如 Feb/、Mar/
    └── <日期>th.md   # 按日期命名的日报，如 7th.md、15th.md
```

## 日报格式

每篇日报建议包含以下部分：

| 部分 | 说明 |
|------|------|
| **Task Point** | 今日完成的工作任务（编号列表） |
| **Difficulties** | 遇到的困难或卡点 |
| **明日计划** | 次日待办或目标 |

## 写法建议

- **Task Point**  
  - 用「动词 + 具体结果」写，方便日后检索（如：读完 MEMIT 论文并做了笔记；梳理完 MEMIT 前向传播流程）。  
  - 尽量写「做到了什么」而不是「在做什么」，便于评估进度。

- **Difficulties**  
  - 写清卡在哪一步、哪一块（如：ROME 的因果追踪部分看不懂）。  
  - 可以顺带写一句「打算怎么解决」（补读 ROME / 问师兄 / 查 issue），后面回顾时更有用。

- **明日计划**  
  - 写 1～3 条具体、可执行的事（如：完成 dataloader 多 GPU 改造），和今天的困难或任务能接上。  
  - 有大目标时，可以拆成「明天做到哪一步」（如：第三天目标：熟悉全流程；第四天：实现迁移）。

- **其他**  
  - 标题可统一为 `Feb 7th work Record`，避免拼写不一致（如 Reword → Record）。  
  - 不必写很长，每条一两句即可，坚持记比写完美更重要。

## 使用方式

1. 在对应月份文件夹下新建 `\<日期>th.md`（如 `Feb/8th.md`）。
2. 按上述格式填写当日工作记录。
3. 便于回顾进度、梳理难点和规划后续工作。
